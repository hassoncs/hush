## Hush defects found

### Defect 1: `hush migrate --from v2` fails with no keys when no `.sops.yaml` exists

**Severity:** Blocks migration path for repos with no prior key setup. Does NOT block all migrations — only those that have never had SOPS initialized.

**Repro steps:**
1. Repo has `hush.yaml` (v2) but no `.sops.yaml` and no matching age key
2. Run `hush migrate --from v2`

**Exact error:**
```
Error: SOPS encryption failed: config file not found, or has no creation rules, and no keys provided through command line options
```

**Expected behavior:** `hush migrate --from v2` should detect the missing key/`.sops.yaml` and either auto-generate a key (like bootstrap does) or emit a clear actionable message: "No age key found. Run `hush keys generate` or `hush bootstrap` first."

**Workaround:** Run `hush bootstrap` first, then `hush migrate --from v2` (which becomes a no-op but doesn't error).

---

### Defect 2: `hush keys generate` fails with "No project identifier found" in legacy-v2 repos

**Severity:** Minor / UX. Does not block migration when workaround is used.

**Repro steps:**
1. Repo has `hush.yaml` with `project: bootstrap`
2. Run `hush keys generate`

**Exact error:**
```
No project identifier found.
Add "project: my-project" to hush.yaml
```

**Expected behavior:** Should read `project` from `hush.yaml` (it is present). The v3-migration-required banner fires first and then the identifier lookup fails, suggesting the keys command is not reading the legacy config for project identity.

**Note:** `hush bootstrap` did successfully auto-detect the project slug as `new-app` (from directory name, not `hush.yaml`).
