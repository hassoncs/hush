## Bug hit: object-keyed targets crash migrate --dry-run

The `hush.yaml` used an object-keyed `targets` format (not an array):

```yaml
targets:
  backend:
    env:
      TORRENTLEECH_PASSKEY: ""
```

`hush migrate --from v2 --dry-run` crashed with:

```
Error: config.targets.map is not a function. (In 'config.targets.map(...)', 'config.targets.map' is undefined)
```

Root cause: `normalizeLegacyTargets` in `hush-cli/src/v3/legacy-v2.ts` assumed `targets` was always an array. The object-keyed format is not standard v2 but does appear in the wild.

**Fix applied** to `/Users/hassoncs/Workspaces/Personal/dev/hush/hush-cli/src/v3/legacy-v2.ts`:
- `normalizeLegacyTargets` now accepts `unknown` and handles both array and object-keyed formats
- Object-keyed entries without `path` or `format` (schema-only declarations) are silently dropped
- All existing tests pass; type-check clean
